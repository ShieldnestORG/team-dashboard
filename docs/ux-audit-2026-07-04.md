# UX Audit — Content Creation & Funnel Setup Flows (2026-07-04)

> **Cluster:** socials/content UX · **Tags:** ux-audit, compose, funnels, content-hub, daily-brief, affordance, dropdowns, button-hierarchy · **Related:** [Socials Hub](products/socials-hub.md), [Funnel Library](products/funnels-library.md), [Daily Brief](products/daily-brief.md), [Marketing Content Hub Handoff](marketing-content-hub-handoff.md)

Owner review of PRs #145–#154 found the content/funnel UI confusing. Complaints:
(1) can't tell dropdowns from buttons, (2) "all the colors are the same",
(3) flows aren't self-explanatory. Bar: **a 10-year-old must be able to use it
without help.**

Method: walked Compose, Funnels (library + draft→live stepper), Content Hub,
Content Studio (Review), Queue, Daily Brief, and Inspiration as a first-time
user. Line numbers are pre-fix positions.

Severity: **HIGH** = blocks a novice · **MED** = slows them · **LOW** = polish.

---

## What is already good (don't touch)

- **Compose**: per-account success/failure rows, platform-gated account chips
  with "needs a photo or video" hints, disabled-with-reason pattern.
- **FlowStepper** (Create→Review→Queue→Posted) on Content Hub, Content Studio,
  and Socials Hub — numbered, plain-English, every step is a link.
- **Funnel lifecycle stepper** (PR #153) — numbered, per-step action buttons,
  disabled-reason tooltips (`GuardedActionButton`).
- **Canonical colors** (`ui/src/lib/status-colors.ts`, PR #152) — statuses and
  platforms are consistent everywhere; all fixes below reuse it, nothing new.
- **Content Hub** kit cards: one filled primary ("Copy the whole kit"),
  labeled expand button with chevron, HelpTips.
- Confirm dialogs on every dangerous funnel action (arm, account-wide off).

## Review round (2026-07-04, post dual review)

Code review verdict MERGE-READY, UX review CLOSE-BUT-GAPS; all raised items
landed on this branch:

1. **Inspiration delete dialog** — Radix Action auto-closed even on failure,
   hiding errors and making the pending label dead code. Now `preventDefault`s
   and only closes via mutation `onSuccess`; failures toast and keep the
   dialog open.
2. **Compose platform copy** — "Instagram and TikTok … photo or video" was
   wrong for TikTok (video required). All three copies (HelpTip, Step 1,
   Step 3) now say Instagram = photo or video, TikTok = video.
3. **Socials Hub tabs** — visual reorder + Compose default landing (see C4).
4. **Queue failed posts** — raw `error: … attempts 3/5` replaced with plain
   English plus a "Try again in Compose" link that carries the post text over
   (`prefillText`); non-failed error states keep the compact line.
5. **Compose submit** — label reads "Pick an account first (Step 1)" when
   nothing is selected; over-limit disable explains itself via tooltip.
6. **Content Studio "Comment" trap** — the note box flags (rejects) on Enter,
   so the button is now "Flag with a note" and the placeholder says "Why are
   you flagging this? Press Enter to flag." (both text + visual cards).
7. **Live-funnel Retire** — now confirms (it deletes the Zernio automation on
   the spot); retiring a merely-ready funnel stays instant since nothing is
   live. Reject on drafts stays instant for the same reason.
8. **PublishTargetChips** — full target names in titles/aria ("CoherenceDaddy
   blog", "ShieldNest", "Tokns app"), retry buttons get tooltips, and failures
   render inline in plain text instead of hiding in a hover title.
9. **Daily Brief "Run now"** — renamed "Write a fresh brief" with an
   explanatory tooltip; success toast no longer claims the brief is already
   visible ("will appear in a moment").
10. **Compose media-URL box** — folded into Step 3 and reworded ("Or paste a
    link to a photo or video that's already online").
11. **Jargon sweep** — "Awaiting Brevo sync" KPI → "Waiting to join the email
    list"; stale "Text box" references now point at the "Write your post" box.
